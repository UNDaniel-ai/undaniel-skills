## impact-analysis.md

### 用途
分析改动的影响范围,避免遗漏。

### 模板

```markdown
# 影响面分析

[← 返回 INDEX.md](INDEX.md)

**创建时间**: YYYY-MM-DD HH:mm

## 改动范围

### 后端改动

**新增文件**:
- `src/auth/login.ts`
- `src/auth/jwt.ts`

**修改文件**:
- `src/models/user.ts` - 增加 password_hash 字段
- `src/routes/index.ts` - 增加认证路由

**删除文件**:
- 无

### 前端改动

**新增文件**:
- `src/components/LoginForm.tsx`
- `src/contexts/AuthContext.tsx`

**修改文件**:
- `src/App.tsx` - 增加认证检查
- `src/routes.tsx` - 增加登录路由

**删除文件**:
- 无

### 数据库改动

**新增表**:
- 无

**修改表**:
- `users` - 增加 `password_hash` 字段

**删除表**:
- 无

### 配置改动

**环境变量**:
- `JWT_SECRET` - JWT 密钥(新增)
- `JWT_EXPIRES_IN` - Token 有效期(新增)

- `config/auth.yml` - 认证配置(新增)

### 直接影响
```

## implementation-plan.md

### 用途
分步骤的实现计划,便于跟踪进度。

### 模板

```markdown
# 实现计划

[← 返回 INDEX.md](INDEX.md)

**创建时间**: YYYY-MM-DD HH:mm

## 实施步骤

### 阶段 1: 后端基础

**目标**: 实现后端认证逻辑

**步骤**:
1. [ ] 安装依赖(jsonwebtoken, bcrypt)
2. [ ] 创建用户模型(增加 password_hash 字段)
3. [ ] 实现密码加密和验证
4. [ ] 实现 JWT 生成和验证
5. [ ] 创建登录 API
6. [ ] 创建登出 API

**预计时间**: 2 小时

### 阶段 2: 前端集成

**目标**: 实现前端登录界面和状态管理

**步骤**:
1. [ ] 创建 AuthContext
2. [ ] 创建 LoginForm 组件
3. [ ] 实现登录逻辑
4. [ ] 实现 token 存储和读取
5. [ ] 实现登出逻辑
6. [ ] 添加路由守卫

**预计时间**: 3 小时

### 阶段 3: 测试

**目标**: 确保功能正常

**步骤**:
1. [ ] 单元测试(后端)
2. [ ] 单元测试(前端)
3. [ ] 集成测试
4. [ ] 手动测试

**预计时间**: 2 小时

### 阶段 4: 部署

**目标**: 部署到生产环境

**步骤**:
1. [ ] 代码审查
2. [ ] 数据库迁移
3. [ ] 环境变量配置
4. [ ] 部署到测试环境
5. [ ] 部署到生产环境

**预计时间**: 1 小时

## 总时间估计

- 开发: 5 小时
- 测试: 2 小时
- 部署: 1 小时
- **总计**: 8 小时

## 关键里程碑

1. [ ] 后端 API 完成
2. [ ] 前端集成完成
3. [ ] 测试通过
4. [ ] 生产部署完成
```

## 模板使用规则

### 何时使用模板

**自动使用**:
- 检测到需要创建某类文档时
- 自动使用对应模板

**用户请求使用**:
- 用户明确要求创建某个文档

### 如何使用模板

1. 选择合适的模板
2. 复制模板内容
3. 填充必填字段
4. 根据实际情况调整内容
5. 在 INDEX.md 中添加链接

### 双向链接

**在新文档开头**:
```markdown
[← 返回 INDEX.md](INDEX.md)
```

**在 INDEX.md 的文档关系图中**:
```markdown
- [design.md](design.md) - 技术设计
```
