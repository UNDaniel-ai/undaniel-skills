## 失败类型判断

| 失败类型 | 特征 | 处理方式 |
|---------|-----|---------|
| 技术性问题 | 语法错误、格式错误、工具问题 | 自动恢复 |
| 业务问题 | 需求理解偏差、设计决策问题 | 用户确认 |
| 不确定 | 无法确定失败类型 | 保守策略（请求用户确认） |

## 恢复策略

### 自动恢复流程（技术性问题）
1. **重试**：立即重试 1 次，失败则修改策略后重试（最多 3 次）
2. **简化**：降低要求（如用文字代替架构图），请求用户确认
3. **跳过**：评估风险，向用户说明，请求确认跳过

### 用户确认流程（业务问题）
1. 说明失败情况和原因
2. 请求用户确认恢复策略
3. 确认 → 执行恢复；不确认 → 回退

### 回退机制
1. 识别问题所在阶段
2. 回退到问题阶段，更新文档
3. 检查后续阶段是否受影响
4. 受影响则更新后续文档，请求重新确认
