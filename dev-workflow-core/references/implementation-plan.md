## 根据流程级别调整详细程度

### 快速流程
- 简化计划（3-5 步）
- 每步包含：目标、文件、任务清单、验收标准

### 标准流程
- 标准计划（5-10 步，含子任务）
- 每步包含：状态、优先级、依赖、目标、文件、任务清单、输入、输出、验收标准

### 完整流程
- 完整计划（10+ 步，含子任务、代码示例、风险应对）
- 每步包含：状态、优先级、依赖、目标、文件、任务清单、输入、输出、验收标准、代码示例、备注

## 跨 Chat 支持

新 Chat 开始时，如果检测到是继续之前的任务：
1. 检查是否存在实现计划文档
2. 读取并显示当前进度（总步骤数、已完成、进行中、待开始）
3. 询问用户："要继续实现哪个步骤？"

## 进度更新机制

每完成一个步骤时，必须：
1. 更新步骤状态（⏸️ → 🟡 → ✅）
2. 更新总体进度统计
3. 记录完成时间
4. 同步更新 INDEX.md
